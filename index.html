Candy

— rocks WebRTC
There are plenty of web-based chat clients out there. None of them is able to do native video conferences.

How to run the demo

  • Make sure you run a browser capable of WebRTC (Currently Chrome Canary or Dev is supported).
  • Enable WebRTC in Chrome Canary/Dev.
  • Open the demo
  • Click on the arrow right of a user (in the userlist) and select "video call"

How it works

  • New technology enable Browsers to access the Webcam and Microphone of the System
  • WebRTC supports P2P Connection via a Javascript API
  • The Jingle Candy Plugin uses the Jingle Strophe.js Plugin to establish the session.
  • Conversion of the SDP (Session Description Protocol) to Jingle is done using the sdpToJingle Javascript Library.

Under The Hood

Candy is built with JavaScript using long poll AJAX requests to emulate bidirectional-streams between client and server following the draft standard XEP-0124 of the XMPP Standards Foundation. By using this standard there are no additional requirements on client-side but having a web browser installed.

Used Libraries

  • sdpToJingle a library for converting SDP Messages to Jingle equivalents
  • Strophe.js a XMPP library for JavaScript
  • jQuery The Write Less, Do More, JavaScript Library
  • mustache.js Logic-less templates with JavaScript
